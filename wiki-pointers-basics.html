Pointers and References

Master memory addressing with Linea's ptr type

Overview

Linea v4.1 introduces pointer support for middle-level memory control. The ptr datatype provides a clean, type-safe way to work with memory addresses.

The Ptr Datatype (Recommended)

Basic Usage

The simplest way to work with pointers in Linea:

var x @ int = 42
var ptr_x @ ptr = x  // Automatically captures address
display "Pointer created"

Multiple Pointers

var a @ int = 10
var b @ int = 20
var ptr_a @ ptr = a
var ptr_b @ ptr = b

Traditional Pointer Syntax

Address-Of Operator (&)

Get the address of a variable:

var x @ int = 99
var ptr @ &int = &x  // Get address with &

Dereference Operator (*)

Access the value at a pointer:

var x @ int = 50
var ptr @ &int = &x
var value @ int = *ptr  // Get value: 50
display value

Pointer Arithmetic

Array Pointers

Access array elements using pointer arithmetic:

var arr @ [int] = [10, 20, 30, 40, 50]
var ptr @ &int = &arr[0]  // Point to first element

display *ptr          // 10 (first element)
display *(ptr + 1)   // 20 (second element)
display *(ptr + 2)   // 30 (third element)

Pointer Loops

var arr @ [int] = [1, 2, 3, 4, 5]
var ptr @ &int = &arr[0]
for i from 0~4 {
    display *(ptr + i)
}

Common Patterns

Swapping Values via Pointers

var x @ int = 10
var y @ int = 20
var ptr_x @ ptr = x
var ptr_y @ ptr = y
// Now x and y are tracked via pointers

Matrix Element Access

var matrix @ [[int]] = [[1, 2], [3, 4]]
var ptr @ &int = &matrix[0][0]
display *ptr  // First element

Memory Safety

Linea manages memory automatically. Pointers are safe because:

  • No dangling pointers: Memory is not freed while pointers exist
  • Bounds checking: Array access is checked at runtime
  • Type safety: Pointer types are checked at compile time
  • Automatic cleanup: Memory is freed when no longer needed
⚠️ Warning: Using pointers requires careful attention. Incorrect pointer arithmetic can cause undefined behavior.

Ptr vs Traditional Syntax

Feature Ptr Type Traditional
Syntax var p @ ptr = x var p @ &int = &x
Ease Simple More control
Auto-reference ✓ Yes ✗ Manual

Best Practices

  • Prefer ptr type: Simpler and safer for most use cases
  • Use traditional syntax sparingly: Only when you need explicit control
  • Document pointer usage: Make intent clear in comments
  • Validate bounds: Check array sizes before pointer arithmetic
  • Avoid deep nesting: Keep pointer chains shallow
